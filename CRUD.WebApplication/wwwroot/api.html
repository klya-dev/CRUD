Мой API - мы компания, постинг статей, у нас два клиента, сайт наш, и наше приложение, также есть доступ для сторонних приложений Т.е у меня должно быть минимум два API (считай контроллера), один для свояков, для собственных продуктов, один для сторонних сервисов

Если по функционалу, то у своих продуктов, это возможность написать комент, поставить лайк, написать статью и тд

А у сторонних: Какой-то сторонний сервис, например, пишет статьи и сразу выкладывает эти статьи через API, на все популярные платформы дзен, ВК, мой клиент. А чо круто, написал один раз много данных по потребностям каждого популярного апи, нажал кнопку и одна статья сразу запостилась у всех и вся Т.е я должен принять все данные по статьи: автора, содержимое и тд Но также я должен проверять существует ли пользователь у меня на сервере, т.к, чтобы выкладывать статьи у меня, должен быть зареган пользователь, т.к от его лица постится статья. Эту часть лора уже будет легко продумать
