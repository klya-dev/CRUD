# CRUD
Идемпотентное RESTful API для публикации статей на технологии ASP.NET Core.
Монолит + микросервис.
Более 53.000 строк кода с XML и Swagger документацией.

Это мой сольный проект, над которым я работаю уже более года (с марта 2025).
Цель: Самостоятельно разработать Web API и внедрить в него, как можно больше актуальных и современных технологий. Использовать лучше практики. В качестве основного источника использовать официальную документацию Microsoft.

Функции:
- Все конечные точки написаны на Minimal API с использованием TypedResult.
- Русская и английская локализация HTTP-ответов через ресурсы.
- HTTP-ответы в формате Problem Details с кастомными полями.
- Версионирование API (v1,v2...).
- Полностью описанная Swagger-документация с кастомными Transformer'ами.
- Применение миграций базы данных при запуске приложения.
- Настройка S3 экосистемы при запуске приложения (создание папок, копирование файлов из архива).
- Логгирование через ILogger Serilog с записью в файл и сохранением этих файлов в S3.
- Логгирование HTTP-запросов с дополнительными настройками по типу: не логгирование запросов "/metrics", скрытие access_token в параметрах запроса (актуально для SignalR).
- Кэширование HTTP-ответов (OutputCache) с использованием Redis'а.
- HybridCache.
- 13 HealthCheck'ов. От проверки подключения к базе до проверки подключения к S3, Prometheus, Telegram, SignalR Hubs и тд.
- 19 Options'ов для настройки сервисов и параметров API в appsetting.json. IOptions, IOptionsMonitor...
- Настройка CORS для клиента, настройка Forwarded заголовков (для прокси).
- Настройка RateLimiter'ов для разных ролей пользователей.
- Центролизованная настройка HttpClient'ов через IHttpFactory (services.AddHttpClient) с политиками повторных попыток (Polly) + для gRPC клиентов тоже.
- Телеметрия с использованием OpenTelemetry и Prometheus + пользовательские метрики приложения (выдача премиума).
- Выделенный микросервис отправки электронных писем через MailRu провайдер, взаимодействие с основным монолитом через gRPC и RabbitMQ.
- Грамотная система очереди отправок писем с повторными попытками и ограничениями, которые можно настраивать через appsetting.json.
- Есть настройка для использования Unix Domain Socket gRPC, вместо обычного варианта (EmailSenderOptions.UseUnixDomainSocketGRPC).
- Публичная статическая папка public в wwwroot, никакой логики не несёт, просто для изучения.
- Собственная реализация аутентификации/авторизации через публичные асимметричные ключи (RSA).
- Поддержка OAuth MailRu (не библиотека), т.е генерация CodeVerifier, CodeChallenge, Nonce и весь codeflow в моих руках.
- Микросервис получает публичные ключи через конечную точку "/.well-known/jwks.json" + умный механизм кэширования и повторые попытки.
- Выдача и отзыв Refresh токенов, в том числе и автоматический отзыв.
- Пользовательский глобальный обработчик событий и middleware'ы.
- Пользовательские конверторы для JsonOptions.
- Пользовательские интерсепторы для gRPC в микросервисе (например, для RPC исключений).
- Система уведомлений через SignalR (любое содержимое через админ-панель, например, "Цена обновилась") + поддержка MessagePack.
- Админ-панель.
- Валидаторы для всех Domain и DTO моделей, реализованы через FluentValidation + дополнительная локализация и кастомные правила.
- Пользовательские фильтры для конечных точек (например, валидация модели, проверка IP-адреса для вебхука оплаты, добавление идемпотентности).
- MySQL база данных с полноценной настройкой сущностей через Fluent API (Entity Framework). Более 50 миграций.
- Наследование (TPT), один-к-одному, один-ко-многим, многие-ко-многим.
- Обработка конфликтов параллельности (гонки) внутри сервисов.
- Проецирование нужных данных, AsNoTracking.
- Отдельный локальный nuget-пакет для S3 интеграции, т.к написанный функционал используется, как в монолите, так и в микросервисе, с целью избежания повторюшкиного кода.
- Постраничные списки (PaginatedList).
- Хэширование паролей, создание API-ключей для премиум пользователей.
- Интеграция с ЮКассой для оплаты Premium'а, создание заказов.
- Специальный клиентский API для премиум пользователей с API-ключами.
- Интеграция с SMS провайдером и Telegram'ом для подтверждения номера телефона через СМС.
- Аватарки пользователей с сохранением в S3, проверка сигнатуры файла через первые байты файла.
- Интеграция с MailRu провайдером для отправки электронных писем (смена пароля, подтверждение почты).
- Проверка вредоносного кода и недопустимых тегов в содержимом публикации HTML.
- Паттерн Result в сервисах, вместо привычного exception-паттерна (управление кодом через исключения).
- Весь функционал покрыт Unit, интеграционными, системными, и нагрузочными тестами + Benchmark, для некоторых участков кода.
- Для тестов используется отдельная база данных (с пересозданием, в некоторых случаях InMemory), отдельные контейнеры Docker образов.

Это основные, интересные функции, которые я вспомнил, пока писал это :)
Понятное дело, какие-то мелочи я упоминать не стал, в счёт их огромного количества.
Не стоит забывать, что все технологии и идеи нужно грамотно и искуссно слепить в один проект, что у меня и вышло.
За время разработки этого проекта у меня сформировался определённый code style, которому я старался соответствовать, что тоже не мало стоит.
Я не писал тяп-ляп, я вдумчиво погружался в новую тему и внедрял в проект, тестировал и комментировал код.

Истории коммитов нет т.к для разработки использовался локальный репозиторий, некоторые коммиты пришлось бы скрывать, а мне лень всё разбирать.
Ну и очевидно, если вы не нашли какие-либо секции в appsettings.json, значит они в UserSecrets :)

Для запуска:
- Настроить UserSecrets.
- Запустить Prometheus, RabbitMQ, Микросервис, Redis.
- И WebApi.

Стек: C#, ASP.NET Core, SignalR, gRPC, MySQL, Redis, Entity Framework Core, RabbitMQ, xUnit, Moq, WebApplicationFactory, BenchmarkDotNet, Монолитная + микросервисная архитектура, FluentValidation, DataAnnotations, OpenTelemetry, Prometheus, S3 (AWS S3), Git, Swagger, User Secrets.
